## Configuration fields of file `pipeline.yaml`

>**task**
>> **Mandatory:** Yes
>> 
>> **Parent:** -
>>
>> **Description:** 
>> 
>> **Values:** Block with fields `id`, `name`, `technique`, `predicts`, `used_datasets`, `evaluation`, `generated_result`.

**Example of block *'task'* with its children. Note that the children are incomplete in this example** 
```yaml
task:
  id: 1
  name: "answer_type_classification"
  technique: "RuleBased"
  predicts:
    [block]
  used_datasets:
    [block]
  evaluation:
    [block]
  generated_result: "result_task_1"
```
---

>**id**
>> **Mandatory:** Yes
>> 
>> **Parent:** task
>>
>> **Description:** Number that identifies the task in the pipeline.
>> 
>> **Values:** Integer
---

>**name**
>> **Mandatory:** Yes
>> 
>> **Parent:** task
>>
>> **Description:** Name of the task. This is the same name given by the person that implemented the task. It is very
>> important as it is used to instantiate the correct task class.
>> 
>> **Values:** String
---

>**technique**
>> **Mandatory:** Yes
>> 
>> **Parent:** task
>>
>> **Description:** Name of the technique. This is the same name given by the person that implemented the technique. It 
>> is very important as it is used to instantiate the correct technique class.
>> 
>> **Values:** String
---

>**predicts**
>> **Mandatory:** Yes
>> 
>> **Parent:** task
>>
>> **Description:** Specifies which set of entries (train, dev, test) should be used in the simulation.
>> 
>> **Values:** Block with fields `predict_train`, `predict_dev`, `predict_test`.
>
>**predict_train**
>> **Mandatory:** Yes
>> 
>> **Parent:** predicts
>>
>> **Description:** Used to set the usage of train entries in the simulation.
>> 
>> **Values:** Boolean
>
>**predict_dev**
>> **Mandatory:** Yes
>> 
>> **Parent:** predicts
>>
>> **Description:** Used to set the usage of dev entries in the simulation.
>> 
>> **Values:** Boolean
>
>**predict_test**
>> **Mandatory:** Yes
>> 
>> **Parent:** predicts
>>
>> **Description:** Used to set the usage of test entries in the simulation.
>> 
>> **Values:** Boolean

**Example of block *'predicts'***
```yaml
predicts:
  predict_train: false
  predict_dev: false
  predict_test: true 
```
---

>**used_datasets**
>> **Mandatory:** Yes
>> 
>> **Parent:** task
>>
>> **Description:** List of datasets that will be used in the task.
>> 
>> **Values:** Block with blocks of `- dataset`
>
>**dataset**
>> **Mandatory:** Yes. At least one if the task should process any information.
>> 
>> **Parent:** used_datasets
>>
>> **Description:** Used to configure a dataset that will be used by the task.
>> 
>> **Values:** Block with fields `name`, `input_fields`.
>
>**name**
>> **Mandatory:** Yes
>> 
>> **Parent:** dataset
>>
>> **Description:** Name of the dataset used in the simulation. 
>> 
>> **Values:** String. Value needs to match the name of a dataset defined in `dataset.yaml`.
>
>**input_fields**
>> **Mandatory:** No
>> 
>> **Parent:** dataset
>>
>> **Description:** Mapping of fields that will be manipulated by the task. It is a key/value mapping where the *key*
>> is the name of the parameter within the task, and the *value* is the result generated by another task that should be
>> mapped to the key. For instance, if the task has a 'query' parameters that should receive the result from task 0, the
>> mapping is `query: "result_task_0"`.
>> 
>> **Values:** Mapping of fields.

**Example of block *'used_datasets'*** (note that the key *query* is a task specific field)
```yaml
used_datasets:
  - dataset:
    name: "QAChave"
    input_fields:
      query: "result_task_0"
  - dataset:
    name: "WikiPassageQA"
    input_fields:
      query: "result_task_0"
```
---

>**evaluation**
>> **Mandatory:** Yes
>> 
>> **Parent:** task
>>
>> **Description:** Configures 
>> 
>> **Values:** Block with fields `should_evaluate`, `type`, `set_usage`, `fields`, `metrics`.

**Example of block *'evaluation'* with its children. Note that the children are incomplete in this example** 
```yaml
evaluation:
  should_evaluate: true
  type: "ValueComparison"
  set_usage:
    [block]
  fields:
    [block]
  metrics:
    [block]
```
---

>**should_evaluate**
>> **Mandatory:** Yes
>> 
>> **Parent:** evaluation
>>
>> **Description:** Indicates if the result generated by the task should be evaluated.
>> 
>> **Values:** Boolean
---

>**type**
>> **Mandatory:** Yes, if field `evaluation` is `true`.
>> 
>> **Parent:** evaluation
>>
>> **Description:** Determines that type of the evaluator that will be instantiated to evaluate the result generated 
>> by the task.
>> 
>> **Values:** String
---

>**set_usage**
>> **Mandatory:** Yes, if field `evaluation` is `true`.
>> 
>> **Parent:** evaluation
>>
>> **Description:** 
>> 
>> **Values:** Block with fields `evaluate_train`, `evaluate_dev`, `evaluate_test`.
>
>**evaluate_train**
>> **Mandatory:** Yes
>> 
>> **Parent:** evaluate_train
>>
>> **Description:** Used to indicate if the train set should be evaluated.
>> 
>> **Values:** Boolean
>
>**evaluate_dev**
>> **Mandatory:** Yes
>> 
>> **Parent:** evaluate_train
>>
>> **Description:** Used to indicate if the dev set should be evaluated.
>> 
>> **Values:** Boolean
>
>**evaluate_test**
>> **Mandatory:** Yes
>> 
>> **Parent:** evaluate_train
>>
>> **Description:** Used to indicate if the train set should be evaluated.
>> 
>> **Values:** Boolean

**Example of block *'set_usage'***
```yaml
predicts:
  evaluate_train: false
  evaluate_dev: false
  evaluate_test: true 
```
---

>**fields**
>> **Mandatory:** Yes
>> 
>> **Parent:** dataset
>>
>> **Description:** Mapping of fields that will be manipulated by the evaluator. It is a key/value mapping where 
>> the *key* is the correct value, and the *value* is the result generated by another task that should be compared 
>> to the correct value. For instance, if the correct value in the resource entry is *answer_type*, it is possible
>> to compare it with *result_task_0* with `answer_type: "result_task_0"`.
>>
>> **Values:** Mapping of fields.

**Example of block *'fields'***
```yaml
fields:
  answer_type: "result_task_0"
```
---

>**metrics**
>> **Mandatory:** Yes, if field `evaluation` is `true`.
>> 
>> **Parent:** task
>>
>> **Description:** Used to configure the 
>> 
>> **Values:** Block with fields `f1_score`.
>
>**f1_score**
>> **Mandatory:** Yes
>> 
>> **Parent:** metrics
>>
>> **Description:**  
>> 
>> **Values:** Block with fields `average`.
>
>**average**
>> **Mandatory:** Yes
>> 
>> **Parent:** f1_score
>>
>> **Description:**  
>> 
>> **Values:** `micro`, `macro`.

**Example of block *'metrics'***
```yaml
metrics:
  f1_score:
    average: "macro"
```
